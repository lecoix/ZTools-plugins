# 哈希计算器 MVP 设计文档

## 1. 插件目标

开发一个 ZTools 哈希计算插件，用于快速计算本地文件或字符串的：

- MD5
- SHA1
- SHA256

第一版只支持以上三种算法，不增加其他 Hash 算法。

插件重点是简单、稳定、低内存占用。文件场景适合用户对下载文件、安装包、压缩包、镜像文件等进行基础完整性校验；字符串场景适合快速计算文本、Token、配置片段等内容的哈希值。

## 2. MVP 范围

### 包含功能

- 在同一个输入框内输入或粘贴字符串计算 Hash
- 在同一个输入区域内拖入文件计算 Hash
- 点击选择文件计算 Hash
- 支持单文件和多文件
- 每个任务显示独立计算进度
- 每个任务显示 MD5 / SHA1 / SHA256 结果
- 支持复制单个 Hash 值
- 大文件计算前提醒
- 用户可在设置页开启/关闭大文件提醒
- 用户可自定义大文件提醒阈值，单位为 GB
- 使用流式读取，避免大文件占用过多内存

### 不包含功能

- 不支持其他 Hash 算法
- 不做 Hash 对比
- 不做批量导出
- 不做历史记录
- 不做文件夹递归
- 不做并发计算
- 不做暂停/取消计算

## 3. 信息架构

插件包含两个主要页面：

- 主页面：统一输入、文件拖拽、文件选择、进度、结果展示
- 设置页面：大文件提醒开关和阈值设置

### 4.2 顶部区域

顶部包含：

- 插件名称：哈希计算器
- 设置按钮

设置按钮点击后进入设置页面。

### 4.3 统一输入区域

统一输入区域支持三种方式：

- 输入或粘贴字符串
- 拖入文件
- 点击选择文件

默认状态：

```text
输入字符串或拖入文件
在这里输入或粘贴字符串；也可以把文件拖到这个输入框区域。
```

拖拽悬停状态：

```text
松开开始计算文件 Hash
```

输入字符串后，点击“计算字符串”开始计算。

选择或拖入文件后，插件先读取文件大小，再根据设置判断是否显示大文件提醒。

### 4.4 任务汇总

选择文件后显示任务汇总：

```text
本次任务：3 项    已完成 1    正在计算 1    等待 1
```

状态包括：

- 总文件数
- 已完成数量
- 正在计算数量
- 等待数量
- 失败数量，如果存在失败文件则显示

### 4.5 结果卡片

每个文件或字符串任务使用独立结果卡片。

卡片内容：

- 任务名称
- 文件路径或字符串长度
- 文件大小或字符串 UTF-8 字节数
- 当前状态
- 当前进度
- 进度条
- MD5
- SHA1
- SHA256
- 每个 Hash 的复制按钮

## 5. 设置页面设计

### 5.1 页面结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 设置                                               [返回]     │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  大文件提醒                                                  │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ 超过阈值时提醒我                           [ 开启 ]     │
│  │                                                        │
│  │ 提醒阈值                                  [ 2 ]  GB     │
│  │                                                        │
│  │ 选择的单个文件或本次总大小超过阈值时，会在计算前提醒。    │
│  └────────────────────────────────────────────────────────┘  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 5.2 设置项

第一版只包含两个设置项：

```text
largeFileWarningEnabled: true
largeFileWarningThresholdGB: 2
```

说明：

- `largeFileWarningEnabled` 控制是否开启大文件提醒
- `largeFileWarningThresholdGB` 控制提醒阈值
- 默认开启提醒
- 默认阈值为 2GB
- 用户可自由修改阈值

### 5.3 阈值输入规则

- 单位：GB
- 默认值：2
- 最小值建议：1
- 最大值建议：1024
- 输入无效时恢复默认值 2
- 使用输入框上下控件递增或递减时，步进为 1GB

## 6. 大文件提醒设计

### 6.1 触发条件

如果开启了大文件提醒，满足任一条件时显示提醒：

- 单个文件大小超过用户设置的阈值
- 本次选择的文件总大小超过用户设置的阈值

### 6.2 提醒内容

```text
本次选择 3 个文件，共 4.8 GB，计算可能需要较长时间。
插件会使用流式读取，不会一次性加载整个文件。

[ ] 不再提醒大文件计算

[取消] [继续计算]
```

### 6.3 用户行为

用户可以选择：

- 继续计算
- 取消
- 勾选“不再提醒大文件计算”后继续计算

如果用户勾选“不再提醒大文件计算”，则更新设置：

```text
largeFileWarningEnabled: false
```

## 7. 计算策略

### 7.1 字符串计算

字符串使用 UTF-8 编码计算 MD5、SHA1、SHA256。

```js
crypto.createHash('sha256').update(text, 'utf8').digest('hex')
```

### 7.2 文件流式读取

大文件必须使用 Node.js 流式读取，不允许一次性读取完整文件。

推荐方式：

```js
fs.createReadStream(filePath, {
  highWaterMark: 4 * 1024 * 1024
})
```

### 7.3 文件三种 Hash 一次完成

每个文件只读取一次，同时计算 MD5、SHA1、SHA256。

```text
文件流
  ├─ MD5
  ├─ SHA1
  └─ SHA256
```

### 7.4 多文件计算

第一版采用串行计算：

```text
file 1 完成
  ↓
file 2 完成
  ↓
file 3 完成
```

原因：

- 降低内存波动
- 降低磁盘压力
- 简化进度展示
- 避免多个大文件同时读取导致性能下降

## 8. 进度设计

每个任务单独维护进度。

字符串任务通常会立即完成，进度从计算中进入 100%。

文件任务进度计算：

```text
progress = 已读取字节数 / 文件总字节数
```

状态变化：

```text
waiting -> hashing -> done
waiting -> hashing -> error
```

每个任务对象建议包含：

```ts
type FileHashTask = {
  id: string
  kind: 'file' | 'text'
  path: string
  name: string
  size: number
  progress: number
  status: 'waiting' | 'hashing' | 'done' | 'error'
  error?: string
  hashes?: {
    md5: string
    sha1: string
    sha256: string
  }
  text?: string
}
```

## 9. ZTools 插件入口

建议提供命令入口：

```json
{
  "code": "file-hash",
  "explain": "计算文件或字符 MD5 / SHA1 / SHA256",
  "cmds": ["hash", "md5", "sha1", "sha256", "文件哈希", "文件校验"]
}
```

如果 ZTools 支持文件输入触发，可增加文件入口：

```json
{
  "type": "files",
  "fileType": "file",
  "label": "计算文件 Hash"
}
```

## 10. 视觉风格

整体风格：安静、可靠、工具感。

设计原则：

- 不做营销页
- 不使用大面积装饰
- 结果优先，字符串和文件共用同一个输入区域
- Hash 使用等宽字体
- 进度条放在每个任务卡片中
- 复制按钮保持轻量
- 错误状态清晰但不刺眼

建议颜色：

- 背景：浅灰白
- 主文本：深灰黑
- 次级文本：中灰
- 进度条：蓝绿色或绿色
- 错误状态：低饱和红色
- 边框：浅灰

## 11. 复制设计

每个 Hash 结果右侧提供复制按钮。

复制成功后按钮状态短暂变化：

```text
[复制] -> [已复制]
```

约 1.5 秒后恢复。

复制内容只包含 Hash 字符串，不附带算法名或文件名。

## 12. 错误处理

需要处理的错误：

- 文件不存在
- 没有读取权限
- 文件读取中断
- 文件大小读取失败
- Hash 计算异常

错误展示在对应文件卡片内，不影响其他文件继续计算。

## 13. 存储设计

设置优先使用 ZTools 本地存储能力。

可存储字段：

```json
{
  "largeFileWarningEnabled": true,
  "largeFileWarningThresholdGB": 2
}
```

如果 ZTools 存储不可用，可降级使用 `localStorage`。

## 14. 第一版验收标准

第一版完成后应满足：

- 可以通过点击选择文件
- 可以通过拖拽选择文件
- 可以在同一输入框输入字符串并计算
- 可以一次选择多个文件
- 可以计算 MD5 / SHA1 / SHA256
- 大文件不会一次性进入内存
- 每个文件有独立进度条
- 每个文件有清晰状态
- 计算完成后可以复制单个 Hash
- 大文件超过阈值前会提醒
- 用户可以关闭大文件提醒
- 用户可以自定义大文件提醒阈值
- 不出现其他 Hash 算法选项
